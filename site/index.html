四步把问题理顺

  1. 1

    讲情况

    用自己的话说明发生了什么，不必先会法律术语。

  2. 2

    补信息

    根据提示补充日期、工资、岗位和关键经过。

  3. 3

    看建议

    查看事实、估算、证据清单和带来源的行动建议。

  4. 4

    确认保存

    只有明确同意后才保存；需要时脱敏导出。

常见劳动问题

你可能正遇到这些情况

先确认时间线和现有证据，再讨论权利范围与行动选择。

01

公司欠薪

梳理应发、实发、发薪日和沟通记录，列出待核对金额。

02

突然被辞退

核对通知方式、理由、时间与工资基数，避免遗漏关键事实。

03

今天要签协议

逐条看清解除原因、金额、付款时间、放弃权利和违约条款。

04

加班没有记录

从考勤、排班、聊天、邮件、门禁和工作成果寻找相互印证。

05

准备劳动仲裁

按请求、事实、证据和来源组织草稿，并标出仍需当地核对之处。

一个虚构、已脱敏的使用示例

示例提问

“我在广州的公司工作了 2 年，公司今天通知我不用来了，也没给书面辞退通知。请帮我梳理事实、补偿估算要补什么信息，以及现在需要保存哪些证据。”

此示例不对应任何真实个人或案件，不构成个案法律意见。
1

事实要点梳理

列出入职日期、岗位、通知时间与方式；把“已确认”和“待确认”分开。

2

权利与补偿估算（参考）

说明所需工资基数、工作年限和解除原因，展示公式与估算状态。

3

证据清单建议

优先保存劳动合同、工资流水、考勤、书面通知和沟通原件。

4

下一步建议

先书面确认公司意思，再按风险与时效安排协商、复核或仲裁准备。

隐私与信任中心

你的材料，由你决定如何处理

插件本身不自动上传、不自动保存，也不会把咨询内容发送给插件作者。

你提交给 Codex、ChatGPT 或另行启用的模型服务的内容，可能按相应服务的规则处理和保留；使用前请查看宿主与模型服务的隐私说明。

保存前说明位置与范围，并请你再次确认。

⌫

删除后核对目录、索引与关联审计记录。

安装到开发环境

从 Codex 开始

C

Codex（规范入口）

添加本项目的市场来源，然后在桌面端完成插件安装。

codex plugin marketplace add 90le/worker-rights-cn --ref main
  1. 运行上面的市场添加命令；该命令只添加 marketplace，并不会直接安装插件。
  2. 继续运行 codex plugin add worker-rights-cn@worker-rights-cn 完成安装。
  3. 也可以重启/刷新 Codex 或 ChatGPT 桌面端，在 Plugins 中选择 Worker Rights CN marketplace 后安装。

只使用上面的 Codex 命令或 Plugins 界面安装；请勿运行来源不明的安装脚本。
