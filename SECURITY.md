# Worker Rights CN 安全政策

## 报告安全漏洞

不要创建公开的安全漏洞 Issue，也请勿在公开 Issue、聊天截图或社交平台中披露尚未修复的漏洞。请使用仓库的 [GitHub Private Vulnerability Reporting](https://github.com/90le/worker-rights-cn/security/advisories/new) 私密提交报告。

报告应使用最小化、脱敏的复现材料，说明受影响版本、环境、影响和复现步骤；不得附带真实劳动者的个人信息、真实个案证据、访问凭据或第三方数据。如果私密报告入口暂不可用，请等待入口恢复，不要改用公开 Issue 披露漏洞。

维护者收到报告后应确认范围、保护报告材料、复现问题、给出修复或缓解方案，并在必要时协调披露。请勿利用漏洞访问他人数据、扩大影响或破坏数据。

## 支持范围

安全修复以仍受支持的最新公开版本为主。开发构建不应被描述为公开稳定版本。插件本身不应自动上传证据或启用遥测；若发现越权保存、隐私数据进入发行包、宿主绕过安全决策或删除回执可伪造，应按高优先级处理。
