# 如何估算补偿

补偿不是只输入月薪就能得到一个确定答案。

以下示例均为虚构，已脱敏，不对应任何真实个人或案件。

## 现在先不要做什么

- 不要把 N、N+1、2N 当成可以随意相加的固定套餐。
- 不要把社保基数或统计工资直接当作补偿上限。
- 不要遗漏奖金、津贴等工资构成。
- 不要把估算当成公司一定会支付或仲裁一定会支持的金额。

## 今天应当保存什么

准备入职日期、解除日期、工资构成、工资流水、劳动合同、解除理由和工作城市。

如果工资波动，尽量准备完整周期的记录。插件会显示采用了哪些月份和哪些假设。

## 当前可能涉及哪些权益

以下虚构输入只用于说明格式：

> 2022 年 3 月入职，2026 年 7 月收到解除通知。近一段时间月平均工资约 18000 元，工作地为深圳。请分别列出输入、公式、可能路径、缺失信息和来源。

合格结果应包含：

- 输入值和时间范围。
- 工龄折算方法。
- 不同解除路径为什么互斥或附带条件。
- 计算结果及舍入方式。
- 当地数值的来源日期。
- 仍需确认的工资组成、解除性质和当地口径。

金额必须标为 `estimate`。当地上限或办理口径不确定时必须标为 `local_verify`。

## 下一步需要补充什么信息

至少确认：实际工作地、入职与解除日期、近一段时间工资明细、公司解除理由、是否已经支付部分款项。

复杂股权、跨境用工、多主体混同或大额争议应标为 `lawyer_review`。
