# 隐私与本地存储

默认策略是：不保存、不上传、最小采集。

以下示例均为虚构，已脱敏，不对应任何真实个人或案件。

## 默认会发生什么

- 案件内容只在当前会话中使用。
- 插件不自动保存案件。
- 插件不自动上传原始证据。
- 插件不自动对外发送文件或消息。
- 公共法律知识与个人案件数据分开存放。

## 明确保存

当你说“保存本案”时，插件必须先展示：

1. 绝对路径，例如 Windows 盘符开头或 Unix 根目录开头的完整路径。
2. 保存范围，包括案件事实、判断、来源锚点和哪些生成物。
3. 将被遮盖或排除的敏感字段。
4. 这是本机写入，不是上传。

只有你对显示的绝对路径和保存范围明确确认后，才允许写入。你没有确认时，流程应停止，当前对话仍可继续。

可以这样说：

> 保存虚构案件前，先给我脱敏预览、绝对保存位置和 scope。没有我的再次确认不要写入。

这里的 `scope` 是保存范围，不是让你学习内部数据结构。

## 脱敏导出

导出前检查姓名、联系方式、证件号、住址、银行账号、健康信息、家庭信息和无关公司资料。

分享包应使用脱敏值。原始文件应继续留在你控制的位置。插件不会自动把导出包发给律师、公司、仲裁机构或任何第三方。

## 删除和删除证明

你可以说：“删除已保存的本案，并给我删除证明。”

删除应覆盖案件目录、案件索引项和关联案件审计记录。公共法律知识库不受影响。

删除证明应说明：目标案件、删除结果、案件目录是否不存在、索引是否无记录、关联审计是否无记录，以及证明生成时间。删除失败时不得声称已经删除。

当前没有面向普通用户的独立删除命令。请通过 Codex 的案件流程发起删除，以便使用同一存储边界和证明检查。

## 你也要做的事

- 输入前先去掉无关个人信息。
- 只提供处理下一步所需的字段。
- 不把真实原件放入公开仓库或公开任务。
- 导出后检查目标目录和分享对象。
- 删除后妥善处理你自行复制到其他位置的副本。
