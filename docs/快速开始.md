# 快速开始

你只需要在 Codex 里描述自己的劳动问题。

不要先研究技能名。也不要先整理一份很长的法律分析。

## 安装和更新

公开包发布后，在 Codex 的插件安装入口选择 `worker-rights-cn` 发布 ZIP。首次安装、重复安装和从 0.1 版更新都应保留你单独确认保存的案件数据。普通卸载只移除插件；清除案件必须再次明确确认。当前公开发布地址尚未由发布者配置，因此状态是 `pending_external`，不要从陌生链接下载同名压缩包。

## 第一次怎么说

以下示例均为虚构，已脱敏，不对应任何真实个人或案件。

> 公司今天让我签一份离职协议。我在上海工作，还没有签。请先告诉我现在该做什么。

插件会先给出四部分：

### 现在先不要做什么

先不要在未读懂的文件上签字。不要在情绪激动时承认不真实的事实。不要删除原始聊天记录。

### 今天应当保存什么

保存你合法持有的合同、工资记录、通知、聊天上下文和相关时间。保留原文件，不只留裁剪后的图片。

### 当前可能涉及哪些权益

插件会把事实、判断和估算分开。它会显示来源和缺失信息，不会承诺结果。

### 下一步需要补充什么信息

通常先问工作地点、入职日期、工资、收到的文件和最后期限。每次只问一组相关问题。

## 看懂六种状态

- `confirmed_fact`：你已经确认的事实。
- `supported_assessment`：现有事实和来源支持的判断。
- `estimate`：需要核对输入的估算。
- `local_verify`：当地规则或数值需要再查。
- `lawyer_review`：高风险内容应让律师复核。
- `out_of_scope`：不属于中国大陆劳动者侧劳动问题。

## 保存不是默认动作

当前对话默认不自动保存，也不自动上传。

如果你想保存，请说：“保存本案，并先告诉我保存到哪里、保存哪些内容。”插件应先显示绝对路径和范围，再等你明确确认。

如果只想要行动清单，可以说：“先不要保存，只给我今天的行动清单。”
