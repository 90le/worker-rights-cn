# 宿主适配规范

Codex 是规范实现。Claude Code、OpenCode 和 OpenClaw 是薄适配。

## 薄适配可以做什么

- 转换宿主清单和路径格式。
- 把宿主生命周期事件映射到共享 hook。
- 启动同一个 stdio MCP server。
- 显示宿主特有的安装、降级和错误提示。
- 传递宿主明确提供的配置。

## 薄适配不能做什么

- 复制案件路由、安全分类或输出复核条件。
- 复制法律来源、城市数值或补偿公式。
- 建立宿主专属用户案件格式。
- 自动上传、保存或发送材料。
- 在适配失败时让 Codex 主插件一起失败。
- 把 hook 当作唯一安全层。

## 共享边界

所有宿主共用：版本号、`worker_rights_cn` 运行时代码、skills、来源数据、MCP registry、隐私策略和回归案例。宿主清单由同一项目 metadata 同步，不手工维护分叉版本。

adapter 输入只包含事件、路径和配置。业务输出仍由核心模块产生，并经过最终输出复核。

## 新增宿主

1. 记录宿主支持的清单、skills、MCP 和 hook 生命周期。
2. 只实现格式映射和 launcher。
3. 明确不支持能力的降级文案。
4. 验证无 MCP 时仍可给纯技能指导。
5. 验证 adapter 不写用户目录、不复制密钥、不创建遥测。
6. 加入 manifest、路径、事件、阻断、降级和端到端 smoke。

如果某宿主无法执行最终 gate，必须标记为降级支持，不能宣称与 Codex 等价。
