# 把劳动法装进 AI：普通人也能用的「劳动维权 Skill」来了

> **导语 / 摘要**
>
> 这是一个可安装到 Codex 等 AI 工具里的开源劳动权益 Skill。遇到欠薪、被辞退、要求签协议、加班记录不全时，用自然语言说清情况，它会先帮你整理事实、提示该留什么证据、拆开金额计算与风险点，并给出下一步清单。它不代替律师、不替你提交材料，但让普通人在关键时刻先把事情讲清楚、做对第一步。

## 前 30 秒：它是什么，什么时候能用

先说最直接的。

Worker Rights CN 是一套可以装进 AI 工具的劳动权益 Skill。你不需要先读懂劳动法，也不用先把材料整理成律师能看的格式。像平时和 AI 对话一样，告诉它发生了什么，它会先帮你把事情排出顺序。

目前，Codex 是主要使用入口。Claude Code、OpenCode 和 OpenClaw 也有对应接入方式，但具体功能和操作可能不同，本文教程以 Codex 为准。

它重点覆盖五类普通人常遇到的情况。每一种都不是“一键给答案”，而是一条先整理信息、再决定行动的路径：先看欠在哪里、为什么要走、协议写了什么、记录还能怎么互相印证，以及仲裁材料还少什么。

> **【结构组件占位：五类场景卡】** 欠薪｜突然被辞退｜今天要签协议｜加班记录不全｜准备劳动仲裁。HTML 使用五栏或分组卡片呈现，不新增图片。

第一次使用，直接复制下面这句话，把方括号换成你的情况：

> 我在【城市】工作，公司今天【发生的事】。请先区分已确认事实和待补充信息，告诉我现在先不要做什么、今天应保存什么，再列出可能权益和下一步。不要保存或上传我的材料。

第一轮回答不会急着给你一个武断结论，而是先把最紧急的行动顺序拆开。

> **【结构组件占位：四段首答流程】** 现在先不要做什么 → 今天应当保存什么 → 当前可能涉及什么 → 下一步补充什么。HTML 使用四步流程卡呈现，不新增图片。

这就是它最核心的价值：不是让 AI 替你作决定，而是把混乱的一小时，整理成可以执行的下一步。

即使事情没有紧急到“今天必须签”，这套顺序也同样适用。欠薪可以先对齐应发、实发和发薪日；加班材料零散，可以先列出手头有哪些记录；准备仲裁，可以先把请求、事实和证据一一对应。先整理，再判断，比一开始就追问“我能不能赢”更稳妥。

## 我为什么做这个 Skill

我叫丘彬彬，在广州做开源产品。

做这个项目的动机很朴素：普通人在劳动关系突然发生变化时，最缺的往往不是更多法条，而是行动顺序。

下午四点被叫进会议室，桌上放着一份六页协议。HR 说今天必须签。你会先查什么？补偿怎么算？聊天记录要不要截图？不签会不会什么都没有？

这种压力下，一长串专业术语未必能帮你。真正有用的第一步，应该是先停一下：哪些话不要急着说，哪些文件不要急着签，哪些合法持有的材料今天就要保留。

我不希望它表现成一个什么都敢断言的角色。我的判断是：越着急，越要把事实、证据、金额和风险拆开；越是影响重大的决定，越要把最终选择交还给本人。

## 一个简明案例：今天签协议，第一轮到底看什么

> **以下为情景演绎，人物、公司和事件均为虚构，不对应任何真实个案。**

下午四点，小林收到 HR 消息：“组织调整，今天把离职协议签一下，补偿已经给到最大诚意。”

会议室里，协议已经打印好。解除原因写着“双方协商一致”，补偿分两个月支付。HR 又补了一句：“今天不签，后面方案可能就变了。”

小林还没签。他只知道自己在广州工作了几年，却说不清工资基数怎么算，也不知道未休年假和奖金是否要单独核对。

他可以这样提问：

> 我实际在广州工作，2021 年 8 月入职。今天 16:00，HR 口头说岗位取消，要求今天签协议；协议写的是协商解除，补偿分两个月支付。我还没有签。请先给我今天的行动清单，不要保存。

第一段，工具会提醒他：不要因为“今天必须签”就立即签字；不要主动写下与真实意思不一致的“个人原因离职”；不要删除原始聊天、邮件或文件；也不要拿走自己无权访问的公司资料。

第二段，是今天要留什么。劳动合同、续签记录、工资入账、个税和社保记录、会议通知、协议全文，以及与 HR 沟通的完整上下文，都值得先在合法范围内保留。只截一句话不够，最好保留前后语境、时间和发送人。

第三段，才是可能涉及什么。解除方式、工资、补偿、未休假、付款期限，都可以分别检查。但这里出现的数字只是基于当前信息的估算，不是公司已经同意的金额，也不是仲裁或法院的结果。

第四段，是还缺什么。实际工作地、入职和通知日期、最近工资构成、协议原文、公司给出的理由，以及是否处于孕期、医疗期、工伤等特殊状态，都可能改变后续分析。

![虚构的当日签署场景](公众号素材/worker-rights-cn-signing-scene.png)

这个案例想说明的不是“小林最后能拿多少钱”。恰恰相反，它想说明：信息不够时，不要让一个看似精确的数字推着你行动。先把今天不能丢的东西留住，再继续补事实。

## 正式开始前，先看清它能做什么

> **【结构组件占位：能做 / 不能做卡】**
>
> 能做：整理时间线和待确认事实；提示优先保存的证据；展示金额算法与假设；检查协议风险；准备沟通、协商和仲裁草稿；附公开出处并标出需当地确认之处。
>
> 不能做：替你签字、提交、上传或联系第三方；自动保存聊天或案件材料；代表律师、政府或任何官方机构；把估算当成确定结果。
>
> HTML 使用左右对照卡呈现，固定放在安装教程前，不新增图片。

如果正在发生人身安全问题、急性健康风险或自伤风险，请先联系当地紧急服务和可信赖的人。劳动材料可以稍后整理。

如果正在工伤就医、被要求立即签字或临近仲裁时效，也不要只依赖一次 AI 对话。先保留原件和时间线，并尽快联系当地人社部门、工会或持证律师。

## 七步教程：从安装到拿到一份可复核的清单

> **【结构组件占位：七步教程时间线】** 安装 → 第一次提问 → 阅读四段回答 → 分组补事实 → 核对公开出处与金额 → 明确保存或导出 → 最终复核。HTML 使用纵向时间线呈现，不新增图片。

### 第一步：安装到 Codex

命令行安装分两步：

```bash
codex plugin marketplace add 90le/worker-rights-cn --ref main
codex plugin add worker-rights-cn@worker-rights-cn
```

第一条只是添加 Worker Rights CN 的 marketplace 来源，不会直接安装；第二条才完成安装。

使用 Codex/ChatGPT 桌面端的读者，也可以重启或刷新后，在 **Plugins** 中选择 **Worker Rights CN** marketplace 再安装。不要从陌生链接下载同名压缩包。

### 第二步：第一次只讲眼前发生的事

不用写法律分析，也不用上传几十张截图。

先说城市、今天发生的事、有没有签字、对方给了什么期限。最后补一句：“不要保存或上传我的材料。”

输入前，先遮掉与下一步无关的姓名、证件号、电话、住址和银行账号。信息越多不一定越好，够回答当前问题即可。

### 第三步：按四段顺序读

先看“不要做什么”和“今天保存什么”，因为这两部分最有时效性。

再看“可能涉及什么”。注意“可能”两个字，不要自动理解成你一定可以拿到某项补偿。

最后看“还要补什么”。如果回答把事实和推测混在一起，可以直接要求重排：哪些是我明确说过的，哪些是根据材料作出的判断，哪些还没有证据。

### 第四步：一次补一组关键事实

第一组可以补地点和日期：实际工作地、公司注册地、入职日、通知日、最后期限。

第二组再补文件：合同、通知、协议名称、公司理由和付款安排。

第三组才补金额和特殊情况：税前税后工资、固定工资与奖金、孕期、医疗期、工伤等。

一次补一组，比不断追加零散聊天更容易发现矛盾。引用文件时尽量保留完整句子和上下文，但不要提供无权访问或与自己无关的公司材料。

### 第五步：让依据和金额都能看懂

可以继续问：

> 请给关键判断附上可核对的公开出处。金额不要只给结果，请列出算法、输入、假设和仍待确认的地方。

这句话很重要。公开出处只能证明一般规则，未必足以证明你的个案结论；金额算得再精确，如果工资口径或解除方式还没确认，也只能叫估算。

地方规定和办事口径可能变化。看到“需要当地确认”时，不是工具没回答，而是在提醒你不要把不确定的事说成确定。

### 第六步：保存和导出必须由你明确决定

默认情况下，插件不自动保存案件，不自动上传原始证据，也不自动发送文件或消息。

如果只是临时梳理，到这里可以停止。

如果确实要保存，可以说：

> 保存本案前，先给我脱敏预览、绝对保存位置和保存范围。没有我的再次确认，不要写入。

你应该先看到完整路径、具体保存内容、哪些敏感信息会被遮盖或排除，再明确同意。看不到这些信息，就不要确认。

需要导出给律师、工会或自己留档时，也不要把整段聊天直接转发。先检查姓名、电话、证件号、住址、银行账号、健康和家庭信息，还要确认附件里没有无关人员或公司的资料。

一句“帮我看看”，不授权长期保存、导出或发送材料。插件的默认行为也不等于宿主完全不处理会话数据；Codex 或其他宿主如何处理数据，仍受各自功能设置与隐私政策约束。

### 第七步：最终行动前，再对一次原件

把最终结果拆成五栏：已确认事实、待补事实、今天行动、金额估算、需要专业复核的事项。

逐项对照原件，检查日期、公司主体、金额、文件名称和引用有没有抄错。任何准备发给公司、仲裁机构或律师的文字，都要由本人先读一遍。

工具给出的最后一段话不是终点。你核对过的事实，才是下一步的起点。

## 为什么不只问通用 AI

通用 AI 当然能解释很多劳动法概念。问题在于，劳动争议很依赖地点、时间、文件和证据。

同一句“公司让我走”，可能是协商解除，可能是公司单方解除，也可能只是一句口头沟通。少一个日期、一个签字主体或一段完整上下文，后续方向都可能不同。

它的重点不是让回答显得更肯定，而是让你能知道每个结论从哪里来、哪些信息还不确定、哪些动作必须由本人决定。

> **【结构组件占位：五点可信卡】** 公开出处可核对｜金额算法看得见｜保存前再次确认｜越权动作先停下｜最终结果对原件。HTML 使用五栏主题卡呈现，不新增图片。

这五点不能让结果自动正确，却能让错误更容易被发现，让你知道哪一步可以继续、哪一步应该停下来找专业帮助。

## 为什么开源，以及不同 AI 工具是什么关系

这个项目以 Apache-2.0 许可开源。

我选择开源，不是要求普通劳动者去读代码，而是希望公开说明、规则和改动，方便别人检查：引用是否过期，算法是否写错，隐私承诺是否前后一致，重要边界有没有变化。

开源不等于天然正确，也不等于有人为个案结果负责。它带来的价值是更容易核对和指出问题，而不是给任何结论盖章。

目前，Codex 是本文的主要使用路径。Claude Code、OpenCode 和 OpenClaw 也接入同一套核心原则，但各自支持的功能和操作方式可能不同。

可以把它理解为：内容目标相同，装进不同 AI 工具后，按钮、步骤和能调用的能力未必完全一样。使用哪一种，就要分别查看对应说明和宿主隐私政策。

## 最后：先把第一步做对

劳动问题发生时，人很容易被一句“今天必须决定”推着走。

这个 Skill 想做的事没有那么夸张：让普通人在关键时刻先停一下，把已经发生的事实讲清楚，把合法持有的证据留好，把金额和风险拆开，再决定下一步。

如果你身边有人正在经历欠薪、突然被辞退、被要求签协议或准备仲裁，可以把这篇文章转给他。

不用先成为法律专家。安装后，复制那条提示词，先问四件事：现在不要做什么，今天保存什么，可能涉及什么，下一步补充什么。
