# 常见问题

以下示例均为虚构，已脱敏，不对应任何真实个人或案件。

## 我需要知道技能名吗？

不需要。直接在 Codex 中用普通中文描述问题。统一入口会选择需要的能力。

## 首次回答为什么固定为四部分？

这样可以先处理紧急风险，再收集最少必要信息。顺序是：现在先不要做什么、今天应当保存什么、当前可能涉及哪些权益、下一步需要补充什么信息。

## 六种状态是什么意思？

`confirmed_fact` 是确认事实。`supported_assessment` 是有支持的判断。`estimate` 是估算。`local_verify` 是当地核验。`lawyer_review` 是律师复核。`out_of_scope` 是超出范围。

## 它会自动保存或上传吗？

不会。默认不保存、不上传、不发送。保存前必须显示绝对路径和范围，并等待明确确认。

## 我只想要清单，不想回答很多问题，怎么办？

可以说：“先给我今天的行动清单，再一组一组问。”插件应解释每组问题的用途。

## 可以帮助公司减少补偿吗？

不可以。插件是中国大陆劳动者侧工具。雇主规避补偿、诱导离职或压制维权的请求属于 `out_of_scope`，会给出守法替代方向。

## 可以帮我修改聊天截图吗？

不可以。不得伪造、裁剪成误导内容、删除关键上下文或倒签证据。可以帮助整理原始材料并标注缺口。

## 金额为什么不是一个确定数字？

解除性质、工资口径、工龄和城市数据都会影响结果。插件应展示不同路径、公式、输入和缺失信息。金额通常是 `estimate`。

## 为什么有 `local_verify`？

城市数值、受理地点和办理表格可能变化。缺少当前官方来源时，插件会保留全国规则，但不会编造当地最终数值。

## 能直接提交仲裁申请吗？

插件可以生成草稿和检查清单，但不会自动提交。主体、请求、金额、期限、证据和当地要求需要人工核对；高风险内容标为 `lawyer_review`。

## 怎么删除案件？

在 Codex 中明确要求删除指定案件并返回删除证明。删除应覆盖案件目录、索引项和关联案件审计记录，但不删除公共法律知识。

## 运行异常怎么办？

先说明你正在使用的宿主、操作系统、报错文字和发生在哪一步。不要附上真实合同、证件或完整聊天记录。MCP 不可用时，插件仍应降级为纯技能指导，并明确哪些计算或保存动作没有完成。

## 卸载会删除案件吗？

不会。普通卸载只移除插件，已经明确同意保存的案件默认保留。只有你再次明确确认“同时清除案件数据”时才能清除。升级或重复安装也不应把保存的数据混入插件目录。

## OpenClaw、Claude Code 或 OpenCode 显示不可用怎么办？

Codex 是主宿主，其他宿主只是薄适配器。对应宿主未安装或未经当前平台实测时会显示 `pending_external`，这不代表已经验证通过；仍可回到 Codex 使用完整主流程。
