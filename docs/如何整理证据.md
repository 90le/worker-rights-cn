# 如何整理证据

证据整理的目标是保留真实、完整、可说明来源的材料。

以下示例均为虚构，已脱敏，不对应任何真实个人或案件。

## 现在先不要做什么

- 不要伪造、改字、补签或倒签。
- 不要只保留裁剪后的截图。
- 不要删除对自己不利但能说明上下文的内容。
- 不要绕过权限下载无关商业秘密或他人个人信息。
- 不要诱导同事作不真实陈述。

## 今天应当保存什么

按“材料—事实—来源—时间”整理：

| 材料 | 想证明什么 | 来源与取得方式 | 原件状态 |
| --- | --- | --- | --- |
| 劳动合同 | 劳动关系、岗位、工资约定 | 虚构劳动者甲本人持有 | 原文件保留 |
| 工资入账记录 | 实际工资和发放时间 | 本人账户导出 | 原文件保留 |
| 完整聊天记录 | 通知内容和沟通过程 | 本人参与的会话 | 未裁剪副本 |

文件名可用日期和类别，不写真实姓名。例如 `2026-07-通知-虚构材料.pdf`。

为每份材料记录：原始文件名、取得日期、取得方式、是否完整、是否包含他人隐私、对应哪项事实。

## 当前可能涉及哪些权益

插件会生成优先级、证据缺口和“由公司控制的材料”清单。

已有原件可以标为 `confirmed_fact`。从材料推导的结论通常是 `supported_assessment`。需要当地取证规则确认的内容标为 `local_verify`。

## 下一步需要补充什么信息

告诉 Codex：

1. 争议事件是什么。
2. 你目前有哪些材料。
3. 哪些材料只能由公司提供。
4. 是否有签字、账号关闭或材料灭失的紧迫风险。
5. 你是否只要清单，还是要整理成案件时间线。

导出前先看：[隐私与本地存储](隐私与本地存储.md)。
