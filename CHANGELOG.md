# 更新记录

## 0.2.0（候选版）

- 改为 Codex 优先的纯插件架构，删除 Web 产品、HTTP 会话层及其依赖。
- 新增面向普通劳动者的统一入口，首答固定提供风险提醒、事实整理、证据行动和下一步。
- 建立不可由宿主适配器绕过的安全决策与输出复核，并对紧急签署、危险请求和来源状态实行失败关闭。
- 增加确定性补偿测算、协议审查矩阵、证据计划和劳动仲裁材料工作流。
- 本地优先保存；保存前展示绝对路径和范围，必须明确同意；支持脱敏导出与可验证删除回执。
- Claude Code、OpenCode 和 OpenClaw 均为薄适配层，领域逻辑只存在于规范核心；Codex 是主支持宿主。
- 修复 Windows 文件锁、并发保存、崩溃恢复、路径边界和残留数据检查。
- 增加 Windows、Ubuntu、macOS 的 CI 契约，以及隔离的安装、升级、卸载、保留与显式清除测试。
- 增加中文快速开始、72 小时行动、补偿测算、协议审查、证据整理、仲裁准备、隐私和常见问题文档。
- 0.1.x 可升级到 0.2.0；普通卸载默认保留用户案件，只有明确确认清除时才删除。

### 发布限制

- 公开发布前必须配置发布者实际控制的官网、隐私、条款和安全页面 HTTPS 地址。
- 公开发布前必须取得 GitHub Actions 的 Windows、Ubuntu、macOS 真实绿色结果。
- OpenClaw 的真实安装与宿主烟测在具备对应宿主环境后完成；缺失时明确降级，不影响 Codex 核心安全。
